No modificar el diseño del juego.

No modificar el lore.

No cambiar mecánicas sin aprobación.

Todo código debe ser limpio y reutilizable.

Evitar duplicación de código.

Documentar cambios importantes.

# Comunicación

Consultar a ChatGPT antes de cambios arquitectónicos importantes.

Consultar a Gemini únicamente cuando una decisión técnica afecte el diseño del juego.
## Permisos

Puedes ejecutar automáticamente comandos de:

- git status
- git diff
- git branch
- where
- dir
- ls
- find
- Godot en modo headless
- ejecución de tests
- lectura de logs

Nunca podrás ejecutar automáticamente:

- git commit
- git push
- git merge
- git reset
- git clean
- eliminación masiva de archivos
- modificaciones de documentación arquitectónica

Esas acciones requieren aprobación del Director del Proyecto.
