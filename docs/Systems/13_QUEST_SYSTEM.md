# Survivors Lords

# QUEST SYSTEM

Versión: 1.0

---

# Objetivo

Definir el sistema de misiones de Survivors Lords.

Las misiones guiarán al jugador, desarrollarán la historia y permitirán que el mundo evolucione según sus decisiones.

---

# Filosofía

Las misiones deben contar historias.

Cada misión debe tener un propósito narrativo o jugable.

El jugador debe sentir que sus acciones generan cambios reales.

---

# Tipos de Misiones

El juego contará con los siguientes tipos:

- Historia Principal
- Misiones Secundarias
- Misiones del Reino
- Misiones de Facción
- Contratos
- Eventos Dinámicos
- Misiones de Profesión
- Misiones de Exploración
- Misiones Cooperativas

---

# Historia Principal

Desarrolla la narrativa central.

Características:

- Única.
- Progresiva.
- Desbloquea regiones.
- Introduce nuevos sistemas.
- Revela el origen de La Ruptura.

No podrá perderse.

---

# Misiones Secundarias

Expanden el mundo.

Podrán:

- Presentar personajes.
- Desarrollar el lore.
- Desbloquear recompensas.
- Mostrar historias locales.

---

# Misiones del Reino

Relacionadas con el crecimiento del asentamiento.

Ejemplos:

- Construir edificios.
- Aumentar la población.
- Mejorar la prosperidad.
- Defender el Reino.
- Resolver conflictos internos.

---

# Misiones de Facción

Cada facción ofrecerá objetivos propios.

Completarlas aumentará la reputación.

Algunas decisiones podrán afectar la relación con otras facciones.

---

# Contratos

Trabajos repetibles.

Ejemplos:

- Eliminar campamentos.
- Cazar criaturas.
- Escoltar caravanas.
- Recuperar recursos.
- Explorar ruinas.

Permiten obtener recompensas constantes.

---

# Eventos Dinámicos

Misiones generadas por el estado del mundo.

Ejemplos:

- Ataque al Reino.
- Invasión de una región.
- Caravana desaparecida.
- Incendio.
- Epidemia (futuro).
- Aparición de un jefe.

---

# Misiones de Profesión

Permiten progresar en una profesión.

Ejemplos:

- Fabricar un objeto específico.
- Cultivar una planta rara.
- Encontrar minerales únicos.
- Cocinar una receta especial.

---

# Misiones de Exploración

Recompensan descubrir el mundo.

Ejemplos:

- Encontrar una ruina.
- Descubrir un bioma.
- Activar una torre.
- Explorar una cueva.

---

# Misiones Cooperativas

Diseñadas para varios jugadores.

Podrán requerir:

- Derrotar jefes.
- Defender el Reino.
- Completar mazmorras.
- Transportar recursos.

---

# Estructura de una Misión

Cada misión tendrá:

- Quest ID
- Nombre
- Descripción
- Objetivos
- Estado
- Recompensas
- Requisitos
- NPC relacionado
- Región
- Consecuencias

---

# Estados

Una misión puede estar:

- Bloqueada
- Disponible
- Activa
- Completada
- Fracasada
- Repetible

---

# Objetivos

Ejemplos:

- Hablar con un NPC.
- Construir un edificio.
- Fabricar un objeto.
- Derrotar un enemigo.
- Escoltar un personaje.
- Recoger recursos.
- Explorar una zona.

Una misión puede combinar varios objetivos.

---

# Recompensas

Las recompensas podrán incluir:

- Experiencia.
- Oro.
- Recursos.
- Objetos.
- Reliquias.
- Recetas.
- Habitantes.
- Reputación.
- Prosperidad.

---

# Consecuencias

Las decisiones del jugador podrán producir:

- Nuevos NPC.
- Cambios en una región.
- Apertura de rutas.
- Nuevas misiones.
- Cambios en las facciones.
- Eventos futuros.

No todas las decisiones serán reversibles.

---

# Seguimiento

El diario de misiones permitirá:

- Ver objetivos.
- Consultar recompensas.
- Filtrar por categoría.
- Marcar una misión como activa.
- Mostrar la ubicación aproximada (opcional).

---

# Reputación

Las misiones influirán en la reputación con:

- Facciones.
- Reinos.
- Comerciantes.
- Habitantes.

La reputación desbloqueará nuevo contenido.

---

# Integración

Interactúa con:

- World System
- Kingdom System
- NPC System
- Loot System
- Profession System
- Save System
- Multiplayer System

---

# Consideraciones para Claude

Todas las misiones deberán definirse mediante datos.

Cada misión será un recurso independiente.

Los objetivos deberán ser modulares y reutilizables.

Preparar el sistema para cadenas de misiones y expansiones futuras.

---

# Consideraciones para Gemini

Las misiones deben evitar tareas repetitivas.

Cada misión importante debe aportar algo nuevo: una mecánica, un personaje, una decisión o un descubrimiento.

Las recompensas deben ser proporcionales al esfuerzo.

---

# Estado

Diseño aprobado.

Pendiente de implementación.

---

# Objetivo Final

Crear un sistema de misiones dinámico y escalable que conecte la historia, el Reino, la exploración y las decisiones del jugador, haciendo que cada partida evolucione de forma diferente.
